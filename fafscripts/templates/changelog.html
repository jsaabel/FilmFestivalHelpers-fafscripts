22-10-15: Behind-the-scenes updates and improvements

  • The Reimbursements script is back to retrieving live currency rates.
  • The tech sheets script will no longer crash if no exact end time is specified for an event.
  • Data retrieved from relation properties is no longer capped to a maximum of 25 relations.
  • Various minor tweaks and fixes.

22-10-12: Tech sheets

  • Added a script that generates tech sheets/schedules.
  • Minor formatting fix for Volunteer schedule script.

22-10-10: Volunteer schedules

  • Added a script that generates schedules for Volunteers.
  • The Reimbursements script will now skip 'paid' items.

22-10-04: Maintenance update: More robust scripts.

  • The hotel list and schedule export scripts have been improved to handle omissions in the Notion databases more gracefully, throwing error messages rather than crashing altogether.
  • Added some more instructions/things to look out for on a number of scripts.
  • Various other tweaks and fixes.

22-10-02: HTML export for Norwegian translations, better WordPress export, issuing passes and more.

  • Added a script that exports raw html of Norwegian translations.
  • Pushing events to WordPress will now add a button that links to the event's Eventive site.
  • Added the ability to issue a free pass from a guest page.
  • Pushing feature film screenings should not work as intended.
  • Various smaller tweaks and fixes.

22-09-25: Virtual screenings, bug fixes and more.

  • Making virtual film screenings should now work as intended and is no longer considered 'experimental'.
  • Pushing films to Eventive will now also set a poster image.
  • Various fixes to catalogue export implemented in past, unannounced updates.

22-09-18: Updated hotel list script.

  • Changed/updated configuration for excel spreadsheet exported through hotel list script according to the hotel's wishes.
  • Various misc. fixes.

22-09-15: More updates to catalogue script.

  • Introduced ability to only export certain short film programmes. Note that this will still export introductory texts for other programmes for now.
  • Logo script will now (correctly) export ALL logos for a given supporter into a subfolder.
  • Various minor tweaks.

22-09-13: Update to catalogue script.

  • Introductionary texts for film programmes will now be retrieved from the first related event, if data source in the Catalogue db is set to 'txt from event'.
  • Fixed a bug that would cause only a single info line to show up on exported document, despite multiple events being assigned.

22-09-11: Pushing events to Eventive, statistics script, experimental scripts.

  • Events can now be pushed to Eventive from their respective site, provided the database analyzer finds no 'errors' (messages in red). Please note that this could not be thoroughly tested because of incomplete databases etc, so always check whether everything worked as intended by reviewing the event on Eventive after it has been created.
  • Similarly, a button will show up to let you create a virtual event for live-streamed events. Again, this is intended as a point of departure and should be checked for correctness.
  • Added a 'stats' script, which writes relevant statistics about various metrics to Notion and lets you track those over time.
  • Added an 'experimental' section for untested scripts. First off is a script that (hopefully..) will help you create virtual screenings.

22-09-04: Pass export, pushing events to WordPress and more.

  • Added scripts to export pass images (both individual and batch export).
  • The catalogue export script now allows to export texts/images separately.
  • Events can now be pushed to WordPress Note that this is limited to name, text and image due to API restraints, so some manual work will still have to be done after having pushed an event.
  • Films that have 'digital approval' property checked will have this reflected on their catalogue page.
  • 'School' property will now appear on exported catalogue pages.
  • ALLCAPS in properties will no longer prevent films from being pushed to Eventive.
  • Fixed a bug that would cause guest pages to crash if a phone number was assigned.
  • Fixed a bug that would cause schedule export script to crash if incomplete travel information was provided.
  • Various visual tweaks and bug fixes.

22-08-28: Pushing to Eventive and WordPress, new config section and more.

  • Added a "Config" page that lets you update all relevant internal databases. NB that this goes hand in hand with a small number of tweaks that have been made to how the Notion databases should be operated. More on this in person.
  • Films can now be pushed to Eventive. If no major issues with the database entry are found, a button will appear on the film's page.
  • Guest biographies can be pushed to WordPress from a guest page. Note that the Norwegian translation will have to be added manually.
  • Re-enabled the Catalogue export script.
  • Various visual tweaks and bug fixes.

22-08-17: Code improvements, more checks and improved database views.

  • Various code improvements that will facilitate future development.
  • Small tweaks to database analyzer.
  • More information in database explorer.

22-08-08: Introducing database explorer, design tweaks, https and more.

  • The new "database explorer" offers a more convenient infrasctructure to find relevant information on films, guests and events. It gives easy access to relevant scripts and is fully integrated with the database analyzer.
  • When viewing a guest in the database explorer, you can now preview the guest's schedule before exporting the Word file (the button for this function will only be shown if the guests has been assigned at least one event).
  • The site has been given an updated look and should work better on mobile devices.
  • Traffic to and from the site is now https encrypted, bolstering security aspects and making the site reachable at www.fafscripts.com. All sub-sites are reachable for logged-in users only.
  • Added some more instructive/informational text.
  • Various bug fixes and under-the-hood improvements.

22-08-04: Transition to 2022 databases, major updates to database analyzer.

  • All scripts will now use the 2022 databases!
  • Added 'Events' and 'Catalog' options to database analyzer.
  • Clicking a message under 'Results' in the database analyzer will now take you to the respective notion page.
  • Fixed a bug that could crash catalogue export.
  • Various minor improvements to logging functionality.

22-08-03: Guest database analyzer.

  • Running the 'database analyzer' on the 'Guests' database is now possible. For now, the automatic checks that are performed focus mainly on logic around dates and times for travel and assigned events.

22-08-01: Reimbursements and better feedback.

  • New script available: Reimbursements/receipts export.
  • More detailed feedback in case of errors/crashes.

22-07-28: First deployment.

  • Added Changelog site.
